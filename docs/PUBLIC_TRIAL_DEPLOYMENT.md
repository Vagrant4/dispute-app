# DISPUTE public trial deployment

Use this when testers are outside your local Wi-Fi or outside Singapore.

## Recommended trial architecture

- Backend: Render web service using `render.yaml`
- Database: SQLite on Render persistent disk for the first external trial
- Email verification: Gmail SMTP app password
- Mobile APK: rebuilt with `EXPO_PUBLIC_API_BASE_URL=<public backend URL>`
- Billing: disabled first; add Stripe Checkout only after the login/register trial is stable

## Required owner accounts and secrets

You own these accounts; do not put secrets in chat.

1. Render account
2. Gmail account with 2-step verification and an app password
3. Later: Stripe account for subscription checkout

## Render environment values

Set these in Render:

```text
NODE_ENV=production
DATABASE_URL=file:/var/data/dispute.db
UPLOAD_ROOT=/var/data/uploads
CLIENT_ORIGIN=https://dispute.app
SERVER_PUBLIC_URL=https://<your-render-service>.onrender.com
JWT_SECRET=<auto-generated strong secret, at least 32 characters>
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=<your gmail address>
SMTP_PASS=<your gmail app password, not normal gmail password>
EMAIL_FROM=Dispute <your gmail address>
STRIPE_BILLING_MODE=disabled
```

## Health check

After deployment, open:

```text
https://<your-render-service>.onrender.com/health
```

Expected result:

```json
{"ok":true}
```

## Build public APK

Use the public URL when building the APK:

```powershell
$env:EXPO_PUBLIC_API_BASE_URL="https://<your-render-service>.onrender.com"
```

Then build the Android release APK and share that APK with testers.
